Round Entry Grammar — Layout-Independent

Both layouts produce the same extracted structure
Semantic roles — stable across all layouts
pc-name-slot action bonus-action result / value condition enemy-section
Atomic entry pattern — one per initiative slot
PC name — action / bonus action , result / condition
The dash after the PC name is the universal delimiter. Everything before it is a name-slot token. Everything after is action/result/condition tokens.
6 PCs in initiative order, arranged left-right then top-down. Initiative slot assignment changes each encounter.

Initiative order is declared once per encounter (not per round) and held constant. Slots 1–3 fill the left column top-to-bottom; slots 4–6 fill the right column top-to-bottom. Enemy actions always appear below the 2×3 grid as a separate section.
6 PCs in initiative order, each on their own row. More space per entry for bonus actions and extended notes.
Rd [N]
1Gold — Move, FB hit 9 | BA: —
2Att — SW miss | BA: Shillelagh
3Asp — Death ✓ | BA: —
4Ash — EB crit | BA: Hex
5Mal — Rage attack | BA: —
6Goli — Sneak 14 | BA: —
Enemy actions: Hit Gold, Ash — 12 dmg each
Switching layouts is a per-session toggle in Session Intake. The glossary and extraction prompt adapt automatically. A | BA: delimiter separates action from bonus action in this layout, giving room for both to be recorded.
Regardless of layout, OCR extraction always produces this structure. The delta analysis engine only ever sees this — never raw grid positions.
// Same output whether notes used grid or linear layout { "combat_round": 10, "layout_used": "grid_2col", // or "linear" — informational only "initiative_slots": [ { "slot": 1, "actor_id": "pc_001", "actor_abbr": "Gold", "action": "Move, Firebolt", "bonus_action": null, "result": "hit", "value": 9, "condition": null }, // ... slots 2–6 ... ], "enemy_actions": [ { "description": "Hit Gold, Ash", "targets": ["pc_001", "pc_003"] } ] }
Layout independence is enforced at the schema level. Adding a third layout in future requires only a new visual description in the OCR prompt — the output schema, the glossary, and the delta engine are unchanged.
